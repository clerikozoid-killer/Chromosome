# Заметки по спецификации и консолидация RFC

## Источник истины
`DBSE v5.0 «Chromosome».txt` — финальная консолидированная спецификация (v5.0). Все
реализационные решения сверяются с ней.

## Отложено: конвертация `.docx` (нет pandoc)

На машине разработки **pandoc не установлен**, поэтому автоматическая конвертация
исходных `.docx` в Markdown на Этапе 0 отложена. Документы-доноры деталей:

- `BSE/SLUC_v2/RFC-0000 … RFC-0018` — формальные RFC (онтология STS, алгебра,
  Model Lattice, Compiler IR, Runtime, Solver Backends, Security, Testing, ...).
- `BSE/STS/*` — параллельный пласт (`algebra_v2`, `model_lattice_v2`,
  `Unified Kernel Spec_v2`, `realize Rust.docx`, ...).
- `BSE/master_tz_axiom_gemini.pdf` и др.

### Как добыть позже (любой из вариантов)
1. Установить pandoc и выполнить пакетную конвертацию `.docx → .md`.
2. Либо извлечь текст через Python (`python-docx`) одноразовым скриптом.

После конвертации: вынести сюда **расхождения** RFC с v5.0 и решения «какой источник
главнее» по каждому спорному месту.

> ⚠️ Примечание: `BSE/STS/realize Rust.docx` указывает, что рассматривался Rust.
> Текущее решение (согласовано) — реализация на **Python** (см. `ROADMAP.md`).

## Открытые вопросы (накапливаются по ходу)
- L1 parser: голые целые в позиции фактора отвергаются (валиден только литерал `1`,
  напр. в `1/s`); это сделано осознанно, чтобы опечатки вида `m3` (вместо `m^3`) не
  проходили молча. См. `dbse/dimensional/parser.py`.
